## 二、学习路线规划（以 Python 项目为核心，学以致用）

你提到已学过 Python 基础但忘了约 30%，这个遗忘率完全正常。以下规划的核心思路是「以项目驱动复习，用产出倒逼输入」，把知识变成真正的手艺。

### 总时间线（建议 6-8 个月，同步为本科课程打基础）

```
┌──────────────────────────────────────────────────────────┐
│  Phase 1      Phase 2       Phase 3        Phase 4        │
│  (1-2月)      (2-3月)       (2-3月)       (持续)         │
│  Python       数据分析      大数据          综合项目      │
│  复习+        +机器学习      生态入门        +持续打磨    │
│  强化         入门           +Linux                       │
├──────────────────────────────────────────────────────────┤
│              英语并行线（每天 20 分钟）                    │
│  背核心词表  →  读简单英文教程  →  看英文教学视频(字幕)    │
└──────────────────────────────────────────────────────────┘
```

### Phase 1：Python 回炉 + 强化（1-2 个月）

**目标**：把忘记的 30% 补回来，同时往「能干活」的方向升级。

| 模块 | 核心内容 | 产出/检验 |
|---|---|---|
| **语法回顾** | 数据类型、控制流、函数、类与面向对象 | 刷完 [Python官方教程](https://docs.python.org/zh-cn/3/tutorial/) 或廖雪峰教程 |
| **文件与IO** | 文件读写、JSON/CSV处理、异常处理 | 写一个批量重命名/格式转换的小工具 |
| **标准库实战** | os、sys、datetime、re（正则）、collections | 做一个「日志文件分析器」，自动提取IP、错误类型等 |
| **爬虫入门** | requests + BeautifulSoup + 基础反爬 | 爬取一个真实网站数据并存入CSV（如招聘网站岗位信息） |
| **虚拟环境与包管理** | venv、pip、requirements.txt | 为上述项目创建独立的虚拟环境 |

> 🎯 **第一个小项目**：写一个「个人学习笔记管理器」命令行工具——支持 Markdown 笔记的创建、搜索、标签分类，数据存本地 JSON/SQLite。这个项目贯穿文件操作、字符串处理、数据结构、数据库（sqlite3模块），做完它，Python 基本功就扎实了。

### Phase 2：数据分析 + 机器学习入门（2-3 个月）

**目标**：从「会写 Python」升级到「能用 Python 处理数据」。

| 模块 | 核心内容 | 产出/检验 |
|---|---|---|
| **NumPy** | 数组操作、广播机制、矩阵运算 | 用 NumPy 实现一个简单的线性回归（手写梯度下降） |
| **Pandas** | DataFrame操作、数据清洗、分组聚合、透视表 | 从 Kaggle 找一个数据集（如泰坦尼克号），完成完整清洗流程 |
| **Matplotlib + Seaborn** | 折线图、柱状图、散点图、热力图、子图布局 | 对清洗后的数据做 5 张以上不同维度的可视化图 |
| **scikit-learn 入门** | 数据预处理、分类/回归/聚类基础算法、模型评估 | 完成一个完整的数据挖掘项目（如房价预测、客户聚类） |
| **Git 版本控制** | 基本操作：init、add、commit、push、branch | 把以上所有代码上传到 GitHub |

> 🎯 **第二个项目**（可做期末课程设计的底子）：**「某领域数据洞察报告」**——选题如「B站热门视频特征分析」「某城市二手房价格预测」「大学生消费行为聚类」。完整经历：爬数据 → 清洗 → 探索性分析 → 可视化 → 简单建模 → 写结论报告。把这个项目放到 GitHub 上，就是一份有说服力的作品。

### Phase 3：大数据生态 + Linux（2-3 个月）

**目标**：接触大数据框架，理解「数据量大到单机搞不定时怎么办」。

| 模块 | 核心内容 | 产出/检验 |
|---|---|---|
| **Linux 基础** | 文件系统、用户权限、常用命令（grep/awk/sed）、Shell脚本 | 在虚拟机/云服务器上完成日常操作，写一个自动化备份脚本 |
| **Hadoop基础** | HDFS原理、MapReduce思想 | 搭建单机伪分布式环境，跑一个 WordCount |
| **Spark/PySpark** | RDD、DataFrame API、Spark SQL | 用 PySpark 处理一个中等规模数据集（几十万行），与 Pandas 对比性能 |
| **数据库进阶** | MySQL 增删改查、索引、查询优化 | 设计一个学生成绩管理系统数据库并编写复杂查询 |
| **Docker 入门** | 镜像、容器、docker-compose | 用 Docker 部署一个 MySQL+Python 环境 |

> 🎯 **第三个项目**：**「实时数据看板」**——模拟场景：用 Python 脚本模拟实时产生日志数据 → 存入 MySQL → 用 Flask/Dash/Streamlit 做一个 Web 看板实时展示图表。这是面试中非常加分的全链路项目。

### Phase 4：综合实战 + C语言补充（持续）

**目标**：把三语言整合起来，并逐步形成自己的技术栈。

| 模块 | 核心内容 |
|---|---|
| **Flask/Streamlit Web 应用** | 把 Phase 2 的数据分析项目做成可交互的网页应用 |
| **C语言** | 指针、内存管理、结构体、文件操作。不必精通，重点是理解底层机制——知道 Python 的列表为什么比 C 数组慢，理解「解释型」和「编译型」的差异 |
| **算法巩固** | LeetCode 刷 50-100 题（Easy为主），用 Python 写，同步培养面试能力 |
| **软技能** | 学会看文档（官方文档 > 二手博客）、学会提问题（GitHub Issues > 随便问人）、学会用 ChatGPT/Claude 辅助学习而非代劳 |

## 三、三个语言的角色分工

```
        ┌──────────────┐
        │   C 语言      │  ← 底层基础，理解计算机怎么「想」
        │   (地基)      │     指针、内存，训练严谨的工程思维
        └──────┬───────┘
               │
        ┌──────┴───────┐
        │   Python      │  ← 主力工具，数据处理的瑞士军刀
        │   (核心武器)   │     数据分析、爬虫、ML、自动化脚本
        └──────┬───────┘
               │
        ┌──────┴───────┐
        │   Linux       │  ← 运行环境，一切代码最终落地的地方
        │   (战场)      │     部署、运维、Shell自动化
        └──────────────┘
```

**优先级**：Python（80%精力） > Linux（15%精力） > C语言（5%精力，理解原理即可）

学校课堂上会系统地教 C 语言和 Linux，你完全可以把主要课外精力砸在 Python 项目上。C 语言跟着课堂走就够用，不要焦虑。

## 五、关于「学以致用」的关键建议

1. **每个模块必须出活**。不是「看完」，而是「做出来一个能跑的东西」。哪怕代码很丑，跑起来就是胜利。
2. **GitHub 是你的简历**。从第一天起就把代码 push 上去，哪怕只是一个小脚本。两年后这就是你找工作的第一张名片。
3. **不要追求学完再开始做项目**。学到一个新技能，立刻想「这个东西能解决我身边的什么问题？」然后马上动手。
4. **Python 的威力在于生态**。不要只学语法，要尽快接触到 Pandas、requests、Flask 这些库，它们才是真正干活的东西。
5. **忘掉 30% 是好事**。说明你已经接触过足够多的内容，只是没巩固。回炉会比你想象中快很多。

## 六、英语薄弱怎么办？编程生存英语指南

你的英语基础（小学水平、词汇量 <100、会读音标）在编程初学者里**非常普遍**，这不是致命短板。编程用到的英语是一个高度受限的子集，有规律可循，不需要四六级水平。

### 6.1 先打破两个心理障碍

**误区一：「英语不好就不能学编程」**

代码的世界里，你真正需要「读懂」的英文只有三类：
- **关键字**（约 35 个，比如 `if` `for` `def` `class` `return`）
- **报错信息**（模式高度重复，看多了就条件反射）
- **函数名/变量名**（可以用拼音，也可以慢慢积累英文命名习惯）

这三类东西，词汇量不到 200 个就能覆盖 90% 的日常编码场景。

**误区二：「学编程之前必须先把英语学好」**

反过来才对——**编程是你学英语最快的方式之一**。因为在代码里碰到的单词会反复出现，每次出现都有具体的上下文，这种「情境记忆」比背单词书效率高得多。你不需要先学到某个水平再开始编程，而是在写代码的过程中顺便把英语推上去。

### 6.3 英语薄弱时的日常生存策略

```
                ┌──────────────────────┐
                │   遇到看不懂的英文    │
                └──────────┬───────────┘
                           │
              ┌────────────┴────────────┐
              ▼                         ▼
      是报错信息吗？              是文档/文章吗？
              │                         │
     ┌────────┴────────┐        ┌───────┴────────┐
     ▼                 ▼        ▼                ▼
  直接复制整句    对着上面的      用DeepL/      找中文教程
  到百度搜索      「高频报错词表」  Google翻译    对照看
                 找关键词       整段翻译
```

**具体做法**：
1. **报错信息**：不要逐词翻译。复制整句粘贴到百度，99% 的报错有 CSDN/博客园的中文解释。
2. **看文档**：用浏览器自带翻译插件（如「沉浸式翻译」），一键中英对照。
3. **写变量名**：初期完全可以写拼音（比如 `xuesheng_list`），逐渐转换成简单英文（`student_list`）。不要因为纠结命名而打断思路。
4. **注释**：全部用中文写。代码是写给机器看的，注释是写给自己和同事看的。

### 6.4 并行英语学习计划（每天 20 分钟，不贪多）

编程英语不需要雅思托福级别的系统学习。以下是「刚好够用」的方案：

| 阶段 | 时间 | 做什么 | 怎么检验 |
|---|---|---|---|
| **第一月** | 每天 20 分钟 | 背诵上面「编程生存英语核心词汇表」，每天 10 个，配合实际代码去看（在编辑器里敲一遍这些词） | 看到 `import pandas as pd` 不害怕了 |
| **第二月** | 每天 20 分钟 | 选一个英文 Python 教程（推荐 [W3Schools Python Tutorial](https://www.w3schools.com/python/)），用翻译插件辅助阅读，重点看代码示例而非文字描述 | 能不看翻译看懂简单的英文代码注释 |
| **第三月起** | 每周 2 次，每次 30 分钟 | 看 B 站英文编程视频（开中文字幕），推荐「Tech With Tim」「Corey Schafer」。过程：先看一遍中文翻译，再看一遍英文原版 | 对 `variable`、`function`、`parameter`、`argument` 这些词不再陌生 |

> 💡 **你「会读音标」是一个被低估的优势**。很多大学生英语过了四级但不会读单词，看文档时嘴里念不出来，单词就永远是抽象符号。你遇到生词可以自己拼读，这对记忆帮助极大。每次遇到新单词，用你的音标能力读三遍，效果远超默看十遍。
